EisenHour 도메인 이해 문서
===========================

1. 개요
-------
EisenHour는 **시간에 따라 변화하는 Eisenhower Matrix 기반 프로젝트 관리 도구**입니다.
기존 Eisenhower Matrix와 차별화되는 점은 **WBS(Work Breakdown Structure)와 결합하여 'Urgent' 값이 동적으로 변화**한다는 것입니다.

2. 핵심 개념
-----------
2.1 프로젝트 구조
~~~~~~~~~~~~~~~~~~
- **프로젝트(Project)**: 최상위 단위로, 여러 개의 에픽을 포함할 수 있음.
- **에픽(Epic)**: WBS(Work Breakdown Structure)의 중심 단위로, 하나 이상의 태스크를 포함함. 에픽 단위에서 마감일(데드라인)을 설정할 수 있음.
- **태스크(Task)**: 실제 실행 단위로, 개별 마감일을 가질 수 있으며 에픽에 종속됨.

2.2 WBS와 에픽 중심 구조
~~~~~~~~~~~~~~~~~~~~~~~~~~~
- WBS는 **에픽(Epic) 중심**으로 구성되며, 에픽 내부에서도 날짜별로 세분화될 수 있음.
- 각 에픽은 **마감일(데드라인)을 가지며, 해당 마감일이 변경되면 하위 태스크(Task)의 Urgent 값에 영향을 미침.**
- 태스크는 에픽의 날짜 범위를 벗어날 수 없으며, 자동으로 조정됨.

2.3 긴급도(Urgent) 및 우선순위(Priority)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- **Urgent 값은 데드라인에 따라 자동 변경됨.**
- **마감일 변경은 Urgent 값에 영향을 주지만, 진행률 증가는 현재 Urgent 값에 영향을 주지 않음.**
- 현재 우선순위(Priority) 변경 이벤트 및 Urgent 값의 변화 로직에 대한 세부 계획은 정해지지 않음.

3. 주요 동작 원리
----------------
3.1 Urgent 값의 자동 변화
~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 시간이 경과함에 따라 태스크의 Urgent 값이 자동으로 변경됨.
- 마감일이 다가올수록 Urgent 값이 상승할 가능성이 있음.

3.2 마감일과 Urgent의 관계
~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 에픽의 마감일이 변경될 경우, 하위 태스크의 Urgent 값도 영향을 받음.
- 태스크의 개별 마감일이 에픽의 범위를 벗어나지 않도록 자동 조정됨.

4. 미정 사항
-----------
- **Priority 값 변경 이벤트**
- **Urgent 값 변화의 세부 로직** (예: 시간에 따른 변화 비율)
- **사용자가 Urgent 값을 직접 수정할 수 있는지 여부**
