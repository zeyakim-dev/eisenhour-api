EisenHour 도메인 모델
========================

1. 주요 엔티티 (Entities)
----------------------

### 1.1 User
- **속성**:
  - **Username**: 사용자 고유의 이름 (값 객체 `Username`으로 다룰 수 있음)
  - **Email**: 사용자 이메일 (값 객체 `Email`으로 다룰 수 있음)
  - **Password**: 비밀번호 (값 객체 `Password`로 다룰 수 있음)
  
- **책임**:
  - 사용자의 기본 정보를 관리
  - 로그인 인증을 처리
  - 자기 프로필을 수정할 수 있는 기능 제공

### 1.2 Project
- **속성**:
  - **Tasks**: 프로젝트에 속한 여러 태스크 (Task와의 관계)
  - **Members**: 프로젝트에 참여하는 멤버들 (User와의 관계)
  
- **책임**:
  - 프로젝트에 속한 태스크 및 멤버 관리
  - 프로젝트의 상태와 진행을 관리

### 1.3 Task
- **속성**:
  - **Deadline**: 마감일 (Task마다 개별적으로 설정)
  - **Urgency**: 긴급도 (마감일에 따라 변화)
  - **Priority**: 중요도
  - **Status**: 진행 상태 (예: 대기, 진행 중, 완료)
  
- **책임**:
  - 프로젝트의 실행 가능한 세부 작업을 관리
  - 마감일과 진행 상태를 통해 우선순위와 긴급도 결정

2. 값 객체 (Value Objects)
----------------------

### 2.1 Username (값 객체)
- **속성**: 
  - **Value**: 사용자 이름
- **책임**: 
  - 고유한 사용자 이름을 관리

### 2.2 Email (값 객체)
- **속성**:
  - **Value**: 사용자 이메일
- **책임**:
  - 고유한 이메일을 관리

### 2.3 Password (값 객체)
- **속성**:
  - **Value**: 사용자 비밀번호
- **책임**:
  - 비밀번호의 유효성 및 암호화 처리
  
### 2.4 Urgency (값 객체)
- **속성**:
  - **Value**: 긴급도 값 (예: 낮음, 중간, 높음)
- **책임**:
  - 마감일에 따라 자동으로 변화하는 긴급도 값을 관리

### 2.5 Priority (값 객체)
- **속성**:
  - **Value**: 중요도 값 (예: 낮음, 중간, 높음)
- **책임**:
  - 태스크의 중요도를 관리

3. 애그리게이트 (Aggregates)
----------------------

### 3.1 Project Aggregate
- **애그리게이트 루트**: `Project`
- **속성**:
  - `Tasks`: 여러 개의 Task를 포함
  - `Members`: 프로젝트에 속한 User들
- **책임**:
  - 프로젝트에 속한 모든 태스크를 관리
  - 프로젝트에 참여하는 멤버들 관리
  - 프로젝트 진행 상태 및 우선순위 조정

### 3.2 Task Aggregate
- **애그리게이트 루트**: `Task`
- **속성**:
  - `Deadline`, `Urgency`, `Priority`, `Status`
- **책임**:
  - 개별 태스크의 상태를 관리하고, 마감일 및 진행 상태를 기반으로 우선순위를 계산

4. 도메인 서비스 (Domain Services)
----------------------

- **Urgency 업데이트 서비스**: 
  - **책임**: 마감일에 따른 Urgency 값의 자동 업데이트를 처리하는 서비스. 이 서비스는 시간 경과에 따라 Urgency를 자동으로 변경하며, 마감일이 다가오면 우선순위가 변경될 수 있도록 합니다.

- **Task 상태 관리 서비스**:
  - **책임**: Task의 상태가 변경될 때 (예: 대기 → 진행 중 → 완료) 상태 전환을 관리하는 서비스

5. 엔티티 간 관계
----------------------

- **User - Project**: 
  - `User`는 `Project`의 멤버로 참여. (1:N 관계)
- **Project - Task**: 
  - `Project`는 여러 개의 `Task`를 포함. (`Task`는 `Project`에 속하며, 1:N 관계)
- **Project - Member**: 
  - `Project`는 여러 `User`를 멤버로 포함하며, 각각은 프로젝트에 참여 (M:N 관계)
- **Task - Epic**: 
  - `Task`는 `Epic`에 속할 수 있음 (Epic이 하위 태스크를 가진다). (1:N 관계)
  
6. 미정 사항
----------------------

- **도메인 서비스**: 여러 도메인 서비스 간의 상호작용 방식, 또는 서비스 로직의 세부 사항은 아직 확정되지 않음.
- **Task와 Project 분리**: `Project`와 `Task`의 볼륨이 커질 경우, 분리될 가능성 있음. 태스크를 별도의 애그리게이트로 분리할 경우, 각각의 책임을 명확히 분리할 필요 있음.
