EisenHour 유비쿼터스 언어 문서
==============================

1. 핵심 용어 정의
----------------
- **프로젝트(Project)**
  - EisenHour에서 관리되는 최상위 단위.
  - 여러 에픽(Epic)과 프로젝트 멤버(ProjectMember)를 포함.
  
- **에픽(Epic)**
  - WBS(Work Breakdown Structure)의 중심 단위.
  - 날짜에 따른 일정 관리가 가능하며, 마감일(Due Date)을 가짐.
  - 여러 태스크(Task)를 포함할 수 있음.

- **태스크(Task)**
  - 실제 실행되는 작업 단위.
  - 에픽에 종속되며, 개별 마감일(Due Date)을 가질 수 있음.
  - 에픽의 날짜 범위를 벗어나지 않도록 자동으로 조정됨.

- **WBS (Work Breakdown Structure)**
  - EisenHour의 일정 및 작업 구조를 정의하는 방식.
  - 날짜에 따른 에픽과 태스크로 구성되며, 전체 프로젝트의 작업 분해를 용이하게 함.

- **Due Date (마감일)**
  - 에픽 및 태스크의 종료 시점을 나타내는 날짜.
  - 마감일 변경 시, 하위 태스크의 긴급도(Urgent)에 영향을 미침.

- **Urgent (긴급도)**
  - 태스크의 긴급도를 나타내는 값.
  - 마감일(Due Date)에 따라 시간이 경과함에 따라 자동으로 변경됨.
  - 마감일이 가까워질수록 Urgent 값이 상승하는 로직이 적용됨.

- **Priority (중요도)**
  - 태스크의 중요도를 나타내는 값.
  - 현재 EisenHour에서는 별도의 Priority 변경 이벤트나 자동 조정 로직이 구체화되지 않음.
  - Urgent(긴급도)와는 별도로, 작업의 중요성을 판단하는 지표로 사용될 수 있음.

2. 용어의 관계 및 사용
---------------------
- **프로젝트**는 여러 **에픽**을 포함하며, 각 에픽은 WBS를 기반으로 날짜에 따라 관리됩니다.
- 각 **에픽**은 하나 이상의 **태스크**를 포함하며, 태스크는 반드시 상위 에픽의 범위 내에서 마감일(Due Date)을 유지합니다.
- **Due Date**는 에픽과 태스크 모두에 적용되며, 변경 시 **Urgent** 값에 영향을 주어 작업의 긴급도를 자동으로 조정합니다.
- **Priority**는 별도의 중요도 지표로서, 긴급도와는 구분되어 사용되며 향후 변경 이벤트나 정책이 구체화될 수 있습니다.

3. 협업 및 멤버 관련 용어
------------------------
- **User**
  - EisenHour 서비스의 회원으로, 개인 프로필(이름, 이메일, 프로필 사진 등)을 관리합니다.
- **ProjectMember**
  - User가 특정 프로젝트에 참여할 때 생성되는 객체.
  - 프로젝트 내에서 게스트, 멤버, 매니저 등의 역할이 부여되며, 역할에 따른 권한이 추후 구체화됩니다.

4. 이벤트 및 상태 용어
-----------------------
- **데드라인 변경 이벤트**
  - 에픽이나 태스크의 Due Date가 변경될 때 발생하는 이벤트.
  - 이 이벤트는 하위 태스크의 Urgent 값 재계산 및 자동 조정과 연계됨.
  
- **태스크 자동 조정 이벤트**
  - 에픽의 날짜 범위를 벗어나는 태스크가 감지될 때 자동으로 Due Date를 에픽 범위 내로 조정하는 이벤트.
  
- **멤버 관련 이벤트**
  - 프로젝트 참여, 권한 변경, 참여 탈퇴 등의 이벤트를 포함하며, 협업 기능(채팅, 코멘트, 알림)과 연계될 예정.
