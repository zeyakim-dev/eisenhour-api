EisenHour Bounded Context 문서
==============================

1. 개요
-------
EisenHour는 **시간에 따라 변화하는 Eisenhower Matrix 기반의 프로젝트 관리 도구**입니다. 시스템은 여러 개의 바운디드 컨텍스트로 나뉘어 있으며, 각 컨텍스트는 독립적으로 동작하며 서로 정의된 방식으로 상호작용합니다. 각 컨텍스트는 특정 도메인 로직을 처리하며, 도메인 간의 명확한 경계를 설정하여 시스템의 복잡성을 관리합니다.

2. 바운디드 컨텍스트 정의
------------------------
### 2.1 태스크(Task) 컨텍스트
- **핵심 기능**: 태스크(Task) 관련 로직을 담당하는 컨텍스트입니다. 여기서 Urgent 값이 자동으로 변경되며, 각 태스크의 Due Date가 관리됩니다. 
- **주요 역할**:
  - 태스크의 상태 관리 (예: '진행 중', '완료' 등)
  - Urgent 값 변경 로직 처리 (시간에 따른 긴급도 변화)
  - Due Date 관리 및 자동 조정 (에픽의 범위에 맞추어 조정됨)
  
### 2.2 프로젝트(Project) 컨텍스트
- **핵심 기능**: 프로젝트 관련 로직을 담당하는 컨텍스트입니다. 프로젝트의 생성, 멤버 관리, 역할 설정, 프로젝트 내 에픽과 태스크에 대한 기본적인 관리가 이루어집니다.
- **주요 역할**:
  - 프로젝트 생성 및 수정
  - 프로젝트 멤버 관리 (User, ProjectMember)
  - 프로젝트 내 에픽 및 태스크 관리
  - 프로젝트 마감일(Due Date) 설정 및 변경

### 2.3 사용자(User) 컨텍스트
- **핵심 기능**: 사용자(User) 관련 로직을 관리하는 컨텍스트입니다. 사용자 프로필, 인증, 권한 관리 등이 이 컨텍스트에서 처리됩니다.
- **주요 역할**:
  - 사용자 등록, 로그인, 프로필 관리
  - 권한 부여 및 관리 (예: 게스트, 멤버, 매니저)
  - 사용자 알림 및 피드백 처리

### 2.4 협업(Communication) 컨텍스트
- **핵심 기능**: 프로젝트 내에서의 협업 기능을 담당하는 컨텍스트입니다. 채팅, 코멘트, 알림 등과 같은 협업 기능을 처리합니다.
- **주요 역할**:
  - 채팅 및 실시간 커뮤니케이션
  - 태스크 및 프로젝트 관련 코멘트 작성 및 관리
  - 알림 시스템 (Urgent 값 변경, 태스크 마감일 변경 등)

3. 컨텍스트 간의 관계
----------------------
EisenHour 시스템 내에서 각 바운디드 컨텍스트는 독립적으로 동작하며, 특정 이벤트나 데이터를 통해 상호작용합니다.

### 3.1 프로젝트와 태스크 간 관계
- **프로젝트(Project)** 컨텍스트는 **태스크(Task)** 컨텍스트와 연결됩니다. 프로젝트에 속한 태스크는 프로젝트의 마감일(Due Date) 변경에 영향을 받으며, 이에 따라 태스크의 Urgent 값이 자동으로 변경됩니다.
- 프로젝트 마감일 변경 시, 관련된 태스크들의 Due Date가 자동으로 조정되며, 이를 통해 Urgent 값이 재계산됩니다.

### 3.2 Urgent 값의 변화와 알림
- **Urgent** 값의 변경은 **협업(Communication)** 컨텍스트와 연계되어 알림을 발생시킵니다. Urgent 값이 변경되면 해당 태스크에 할당된 사용자에게 알림이 발송되고, UI에서는 태스크가 하이라이트 처리됩니다.
- 이 변경 이벤트는 태스크의 Urgent 값이 자동으로 업데이트될 때 발생하며, 프로젝트 멤버들에게 중요한 정보로 전달됩니다.

### 3.3 EDA(Event Driven Architecture)
- EisenHour는 **이벤트 기반 아키텍처(EDA)**를 채택하여 각 컨텍스트 간의 상호작용을 이벤트로 처리합니다.
- 예를 들어, 태스크의 Due Date 변경 시, 해당 이벤트는 프로젝트 컨텍스트로 전달되어 Urgent 값이 재계산됩니다. 이후 알림 시스템이 해당 변경을 감지하고 협업 컨텍스트에서 알림을 처리합니다.

4. 데이터 흐름 및 경계
------------------------
### 4.1 프로젝트 및 태스크 데이터 흐름
- 프로젝트와 태스크의 데이터를 처리하는 각 컨텍스트는 독립적인 데이터베이스를 가질 수 있습니다. 각 컨텍스트는 이벤트를 통해 다른 컨텍스트와 데이터를 공유하거나 업데이트합니다.
- 예를 들어, 태스크의 Due Date가 변경되면 **프로젝트(Project)** 컨텍스트에서 해당 태스크를 찾고, **Urgent** 값을 계산하여 **협업(Communication)** 컨텍스트로 이벤트를 발생시킵니다.

### 4.2 알림 및 UI 반영
- 태스크의 Urgent 값 변경 이벤트는 **협업(Communication)** 컨텍스트에서 알림으로 전파되며, UI에서는 해당 태스크를 하이라이팅하여 사용자가 즉시 확인할 수 있도록 합니다.
