EisenHour 애그리게이트 경계
=========================

1. 애그리게이트 루트: 프로젝트 (Project)
-------------------------------
**프로젝트(Project)**는 시스템에서 가장 상위에 위치하는 애그리게이트 루트입니다. 프로젝트는 여러 개의 **에픽(Epic)**과 **프로젝트 멤버(ProjectMember)**를 포함하며, 프로젝트 전반에 대한 로직과 기능을 관리합니다. **프로젝트** 애그리게이트는 **에픽**을 생성, 업데이트 및 삭제하고, **프로젝트 멤버**의 참여를 관리하는 역할을 합니다.

- **애그리게이트 루트**: `프로젝트(Project)`
- **포함된 애그리게이트**:
  - `에픽(Epic)`
  - `프로젝트 멤버(ProjectMember)`

`프로젝트` 애그리게이트의 책임:
- `에픽` 애그리게이트를 생성하고 관리함.
- `프로젝트 멤버` 애그리게이트를 추가 및 삭제함.
- 프로젝트의 전체 정보(예: 시작일, 종료일, 진행률, 우선순위 등)를 유지 관리함.

2. 애그리게이트 루트: 에픽 (Epic)
---------------------------
**에픽(Epic)**은 **프로젝트** 안에서의 하위 애그리게이트로, 프로젝트 내 특정 목표나 기능을 나타내는 큰 단위입니다. 각 **에픽**은 여러 개의 **태스크(Task)**를 포함하며, 태스크는 각각 개별적으로 추적할 수 있습니다.

- **애그리게이트 루트**: `에픽(Epic)`
- **포함된 애그리게이트**:
  - `태스크(Task)`

`에픽` 애그리게이트의 책임:
- **마감일(Deadline)**을 유지하며 관리함.
- **태스크(Task)** 애그리게이트를 포함하고, 해당 태스크들을 관리함.
- **에픽**의 마감일에 따라 **태스크**의 긴급도(Urgent)가 자동으로 변경됨.

다른 애그리게이트와의 관계:
- **에픽**은 **프로젝트**에 속하며, 프로젝트의 전체 계획에 영향을 미침.
- **에픽**의 마감일은 하위 태스크들의 Urgent 값에 영향을 주며, 이를 통해 **태스크**들의 우선순위가 조정됨.

3. 애그리게이트 루트: 프로젝트 멤버 (ProjectMember)
--------------------------------------------------
**프로젝트 멤버(ProjectMember)**는 프로젝트에 참여하는 **사용자(User)**를 나타내는 애그리게이트입니다. 각 **프로젝트 멤버**는 프로젝트 내에서 특정 역할을 수행하며, 프로젝트의 진행 상황에 참여하고 기여합니다.

- **애그리게이트 루트**: `프로젝트 멤버(ProjectMember)`
- **포함된 애그리게이트**:
  - `사용자(User)` (프로젝트 멤버는 특정 사용자를 참조함)

`프로젝트 멤버` 애그리게이트의 책임:
- 프로젝트에 대한 참여 및 권한을 관리함.
- 프로젝트의 진행 상황에 대한 알림을 받으며, 필요한 경우 프로젝트 작업에 대한 코멘트를 달 수 있음.
- 프로젝트 내에서 **태스크(Task)**를 할당받을 수 있으며, 이를 수행함.

4. 애그리게이트 관계
-------------------
- **프로젝트(Project)**는 **에픽(Epic)**과 **프로젝트 멤버(ProjectMember)**를 포함하는 루트 애그리게이트입니다.
- **에픽(Epic)**은 **태스크(Task)**를 포함하며, **태스크**는 **에픽**의 마감일에 따라 Urgent 값이 변경됩니다.
- **프로젝트 멤버(ProjectMember)**는 **사용자(User)**를 참조하며, 각 사용자는 프로젝트에서 특정 작업을 할당받고 진행합니다.
